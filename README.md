# 企业专利全生命周期管理系统 - 演示版

纯前端演示，无需服务器，双击 `demo.html` 即可在浏览器打开，输入任意密码即可进入。

---

## 项目概述

企业专利全生命周期管理系统，是一套面向企业专利管理人员的一站式工具，覆盖专利从**撰写 → 申请 → 审查 → 授权 → 维护 → 终止**的全流程管理。核心解决专利年费逾期、状态跟踪混乱、数据分散等痛点。

---

## 核心亮点

- **全生命周期跟踪** — 12 种权利状态流转（撰写中→已申请→形式审查中→……→专利权生效→已终止），每个状态配有对应的待办事项和附件要求
- **自动费用计算** — 根据专利类型、费减比例、年度自动计算应缴年费、申请费、授权登记费等，逾期自动预警
- **状态流转指引** — 每个状态到下一状态的操作步骤 + 所需附件 + 费用一目了然，用户按指引操作即可
- **仪表盘可视化** — ECharts 展示资产构成饼图 + 动态存量柱状图 + 费用预算查询
- **Excel 批量导入/导出** — 支持 .xlsx 模板导入，自动去重校验，也支持按字段导出
- **AI 智能生成** — 独立页面，支持两种任务：
  - **生成 IPC 分类号**：根据摘要/说明书/交底书推断国际专利分类号，含置信度评估（高/中/低）
  - **生成摘要**：根据说明书或交底书自动提炼摘要
- **LLM 对接** — 支持云端大模型 API（DeepSeek / 通义千问 / OpenAI），通过 `config/llm.json` 配置，API Key 从环境变量读取
- **数据安全** — 应用密码锁 + 密保问题找回 + 多次失败冷却机制
- **数据备份与恢复** — 一键备份 + 保留最近 5 次快照 + 回收站 30 天自动清理
- **支持两种运行模式** — 可打包为桌面 EXE（Electron），也可作为 Web 服务部署

## 技术栈

| 类别 | 技术 | 用途 |
|------|------|------|
| **运行时** | Node.js | 后端运行环境 |
| **Web 服务** | Node.js 内置 `http` 模块 | 静态文件 + REST API |
| **桌面壳** | Electron | 打包为 Windows EXE |
| **数据库** | sql.js (SQLite WebAssembly) | 本地数据库，无需安装 |
| **大模型** | OpenAI 兼容 API | IPC 分类 / 摘要生成（可配置） |
| **前端** | 原生 HTML + CSS + JavaScript | 无框架，轻量高效 |
| **图表** | ECharts 5 | 饼图、柱状图 |
| **Excel** | SheetJS (xlsx) | 导入/导出 .xlsx |
| **密码** | Web Crypto API (SHA-256) | 前端密码哈希 |
| **打包** | electron-builder + pkg | EXE 打包 |

## 智能生成页使用说明

侧边栏导航 → **智能生成**，支持两种任务：

### 生成 IPC 分类号
1. 选择任务 **"生成 IPC 分类号"**
2. 在三个文本框中分别粘贴：**摘要** / **说明书** / **交底书**（至少填一个）
3. 点击 **"开始生成"**
4. 结果包含：IPC 分类号、判断依据、置信度（高/中/低）+ 置信度理由
5. 可点击 **准确度星星** 评分（1-5 星）

### 生成摘要
1. 选择任务 **"生成摘要"**（摘要框自动隐藏）
2. 在 **说明书** / **交底书** 文本框中粘贴内容
3. 点击 **"开始生成"**
4. 结果展示生成的摘要文本

### 生成历史
所有生成记录自动保存到 **数据中心 → AI 生成历史**，最多保留 10 条。

## LLM 配置说明（正式版）

正式版（`node server.js`）需配置大模型 API：

1. 编辑 `config/llm.json`：
```json
{
  "endpoint": "https://api.deepseek.com/v1/chat/completions",
  "model": "deepseek-chat",
  "temperature": 0.1,
  "maxTokens": 1024,
  "timeout": 30000
}
```
2. 设置环境变量（API Key 不保存在文件中）：
```powershell
set LLM_API_KEY=sk-你的key
node server.js
```
3. 支持任意 OpenAI 兼容接口（DeepSeek / 通义千问 / OpenAI / 本地模型）
